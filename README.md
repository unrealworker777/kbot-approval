# kbot-approval — ассистент-черновик с одобрением

Помогает Константину быстро отвечать в Telegram: следит за постами/чатами,
готовит черновик ответа в его стиле через Claude, но **ничего не отправляет
без личного одобрения**. Реакции ставятся автоматически (низкий риск),
комментарии, ответы в чатах и личные сообщения — только после нажатия
«✅ Одобрить» (или после присланной замены текста).

Личка обрабатывается **только реактивно** — когда человек сам написал.
Никакого автоматического захода в чужие ЛС нет.

## ⚠️ Хостинг: только VPS, не GitHub Actions

Это always-on юзербот с постоянной сессией Telethon и логином в реальный
аккаунт. GitHub Actions для него **не подходит** (нет постоянного процесса,
таймауты). Крутить на VPS (например Timeweb Cloud) под `systemd` или `screen`.
GitHub — только для хранения кода.

## Как это работает

1. **Юзербот** (`userbot.py`, Telethon, логинится в аккаунт Константина) следит за:
   - новыми постами в отслеживаемых каналах → ставит реакцию сразу + готовит
     черновик комментария;
   - сообщениями в отслеживаемых чатах, которые являются **ответом** на
     сообщение самого Константина → готовит черновик ответа;
   - входящими личными сообщениями (по правилу `DM_HANDLING`) → готовит черновик.
2. Черновик генерирует **Claude** (`draft.py`) голосом Константина на основе
   `style/style_profile.md`, который ты один раз собираешь из его реальной переписки.
3. **Бот-одобрятор** (`approval_bot.py`, отдельный бот через @BotFather) шлёт
   в личку: оригинал + черновик + кнопки **✅ Одобрить / ✏️ Заменить / 🚫 Пропустить**.

**Ни одно сообщение не уходит собеседнику, пока Константин лично его не увидит
и не нажмёт кнопку.**

## Установка

```
pip install -r requirements.txt
cp .env.example .env
```

Заполни `.env`:

1. **`TELEGRAM_API_ID` / `TELEGRAM_API_HASH`** — https://my.telegram.org/apps
   под аккаунтом Константина.
2. **`TELEGRAM_PHONE`** — номер аккаунта Константина (с `+`).
3. **`APPROVAL_BOT_TOKEN`** — @BotFather → `/newbot`. Это ОТДЕЛЬНЫЙ бот.
4. **`APPROVAL_CHAT_ID`** — свой numeric ID у @userinfobot; один раз напиши
   новому боту в личку.
5. **`ANTHROPIC_API_KEY`** — можно пусто, если сделан `ant auth login`.
6. **`MONITORED_CHANNELS` / `MONITORED_CHATS`** — юзернеймы без `@`, через запятую.
   Пустой список = не слушать (раньше пустой ошибочно слушал всё).
7. **`DM_HANDLING`** — `non_contacts` (по умолчанию: только холодные входящие,
   переписки с контактами в API не уходят), `all` или `off`.

## Собрать голос Константина (обязательно перед первым запуском)

Без этого бот пишет нейтральным тоном (заглушка в `style/style_profile.md`).

1. Собери примеры реальных сообщений Константина:
   - **Экспорт Telegram**: Telegram Desktop → чат → ⋮ → «Экспорт истории» →
     JSON → `result.json`, затем:
     ```
     python style/convert_telegram_export.py result.json "Имя как в Telegram" style/examples/telegram_1.txt
     ```
   - **Вручную**: вставь 20–50 его сообщений в `style/examples/manual.txt`.
   - **Транскрипты уроков**: распознай в текст и положи `.txt` в `style/examples/`.
2. Запусти:
   ```
   python style/build_style_profile.py style/examples
   ```
   Появится `style/style_profile.md`.

## Запуск

```
python main.py
```

Первый запуск: Telethon попросит код из Telegram (придёт на аккаунт Константина).
Сессия сохранится в `konstantin_session.session` (в `.gitignore`, не коммитится).

## Что поправлено в этой версии

- **Комментарии к постам** теперь уходят в привязанную **группу обсуждений**
  (ответом на форвард поста), а не в сам канал — иначе не публиковались.
- **Карточка одобрения** экранирует HTML и обрезает длинный контекст — раньше
  падала на символах `< > &` и длинных постах.
- Генерация черновика вынесена в поток (`asyncio.to_thread`) — не блокирует
  event loop.
- Модель черновиков — **Haiku** (быстро и дёшево для потока коротких ответов).
- **ЛС**: `DM_HANDLING=non_contacts` по умолчанию — контент личных переписок с
  контактами не уходит в API.
- Пустой список каналов/чатов больше не означает «слушать всё».
- В очередь добавлен TTL, ошибки отправки показываются в карточке.

## Важно понимать

- **Реакции ставятся автоматически**, без одобрения. Хочешь и их через очередь —
  скажи, вынесу.
- **Юзербот-автоматизация формально не приветствуется правилами Telegram.**
  Одобрение каждого текста снижает риск (не спам-паттерн), но при аномальной
  активности аккаунт может получить ограничения. Не гони объёмы.
- Очередь одобрения — **в памяти процесса**; перезапуск теряет неотвеченные
  черновики (сами посты/сообщения никуда не денутся).
- `style_profile.md` — только манера речи, без личных данных третьих лиц.

## Настройка

- Промпт черновика — `draft.py`, `SYSTEM_TEMPLATE`.
- Модель — `DRAFT_MODEL` в `.env`.
- Каналы/чаты, эмодзи, режим ЛС — в `.env`.
